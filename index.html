MindBridge

심리상담 초기 접수면접을 AI 인터뷰로 구조화하고, 상담사용/사용자용 리포트와 상담 자원 후보 탐색까지 연결한 LangChain 기반 PoC 프로젝트입니다. 핵심은 상담을 대체하는 것이 아니라, 접수면접과 정보 정리 업무를 보조하는 것입니다.

3 Weeks 미니 프로젝트 기간
4 Chains Interview / Extractor / Crisis / Report
POC 접수면접 · 리포트 · 자원 탐색

Project Problem

상담센터의 초기 접수면접은 반복 질문, 기록 부담, 위기 응답 표시, 후속 자원 탐색이라는 여러 업무가 한 번에 겹치는 구간입니다. MindBridge는 이 구간을 AI로 완전히 대체하는 것이 아니라, 상담사가 더 빠르게 핵심 정보를 파악하도록 돕는 구조로 설계했습니다.

접수면접 부담

동일하거나 유사한 질문 반복, 자유서술 답변 정리 부담, 고정 문진의 한계가 존재합니다.

역할 경계

AI가 상담사처럼 보이거나 임상 판단을 하는 것처럼 보이지 않도록 안전선을 설정했습니다.

자원 탐색

상담센터와 상담사 정보가 흩어져 있어 조건 기반 후보 탐색 보조 기능을 설계했습니다.

Architecture

하나의 거대한 프롬프트가 아니라, 기능별 역할을 분리한 LangChain 파이프라인으로 설계했습니다.

Pipeline

  • Interview Chain: 9섹션 기반 질문 생성
  • Extractor Chain: 자유서술 응답 구조화
  • Crisis Chain: 위기 응답 보수적 분기
  • Report Chain: 사실 정리 중심 리포트 생성

Tech Stack

  • Streamlit UI
  • LangChain / LCEL
  • JSON / CSV 구조화 데이터
  • Debug Layer / Prompt Guardrail

Key Design Decisions

1. 상담 챗봇이 아니라 접수면접 보조 시스템으로 범위 제한 AI가 공감 상담자처럼 보이지 않도록 역할을 정보 수집과 구조화로 제한했습니다.
2. 운영 판단과 임상 판단 분리 질문 진행, 보완 질문, 다음 섹션 이동은 운영 판단으로 보되, 진단과 치료 방향 제시는 금지했습니다.
3. 자원 탐색은 추천이 아니라 후보 안내 센터와 상담사 후보를 조건 기반으로 보여주되, 최종 매칭이나 적합성 단정은 하지 않도록 설계했습니다.
4. 디버그는 새 기능이 아니라 관찰 레이어 질문 흐름은 유지하고, intent, classification, reason, action을 확인하는 방식으로 검증 가능성을 높였습니다.
